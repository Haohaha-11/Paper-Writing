# Docs

This folder stores research and design rationale. The operational project surface is:

- `skills/` for agent-readable writing skills;
- `venues/` for official-source venue rules and checklists;
- `workflows/` for repeatable paper-writing processes;
- `templates/` for template provenance;
- `examples/` for reusable artifacts.

## Key Research Notes

- `repository-evaluation.md`: first-pass upstream project evaluation.
- `second-round-research.md`: official venue/template/TMI/parser gap analysis.
- `writing-skills-research.md`: writing skill ecosystem and project-owned skill taxonomy.
- `cspapers-integration.md`: cspapers.org integration plan for novelty and related-work review.
- `latex-parser-options.md`: future parser/tooling options.
- `venue-rules.md`: venue rule storage model.
- `template-registry.md`: template registry model.
- `tmi-notes.md`: IEEE TMI-specific notes.
- `tex-environment.md`: installed TeX environment and compile validation.

## Rule

When operational files and research notes disagree, prefer the current operational files in `skills/`, `venues/`, `workflows/`, `templates/`, and `examples/`, then update the docs to match.
